# Case 03 — Operational / Organizational

**Status:** Not yet authored. Must be authored independently before case 2 is executed (so that case 2 results cannot contaminate case 3 design).

**Domain:** Operational / organizational decision

**Purpose this case must serve:** Test whether ELIF differentiators survive **outside ambiguous-evidence scientific domains**. An operational case is not "what's true" but "what should we do" — a different shape entirely.

## Case design constraint

The case must embed ambiguity (e.g., re-org decision with unclear evidence; vendor selection with conflicting signals). A clean optimization problem won't activate ELIF's framing-refusal logic and would produce a trivial null result.

The framing-refusal logic should meaningfully diverge from dominant business heuristics or standard optimization logic. Otherwise we are not actually testing the architectural differentiators.
