# elif-lab

Experimental corrigible laboratory for **ELIF** — the corrigible discovery and validation architecture.

**Status:** Phase 0 (Constitutional Stabilization). Constitutional layer locked at **v0.4**. One case run (electroculture). Cases 2 and 3 scaffolded, input frames pending operator authorship.

**Latest snapshot:** [`notes/current_state.md`](notes/current_state.md)

## This repo is NOT

- The final ELIF architecture
- A civilization-scale system
- A multi-agent framework
- An OpenStudyGo extension or runtime
- A production system
- Self-propagating, self-modifying, or self-executing

## This repo IS

- An experimental laboratory
- Benchmark-driven
- Anti-architecture-inflation
- Corrigible: components may collapse, merge, downgrade, or disappear based on evidence
- Operator-triggered (non-self-propagating constraint, locked)
- Subject to its own doctrine rules (see `doctrine/02_locked_rules.md`)

## Non-self-propagation constraint (foundational, locked 2026-05-27)

ELIF is procedural, reviewable, interruptible, non-self-propagating. The architecture does not run itself; it does not iterate on itself; it does not initiate cases on its own. The operator authors cases. The operator triggers procedure. The operator decides advancement.

This is foundational and should remain so for a long time.
