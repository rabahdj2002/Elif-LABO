# Separation from OpenStudyGo

ELIF inherits some **governance principles** from OpenStudyGo, but not **runtime assumptions**.

## What ELIF inherits

| OpenStudyGo principle | ELIF equivalent |
|---|---|
| Anti-fake mastery | Refusal of decorative or self-validating output |
| Observability | Structural transparency of intermediate reasoning states |
| Refusal of decorative completion | Frame refusal; refusal-based governance |
| Strict validation | Hypothesis validator; predictive failure conditions |
| Stabilization discipline | Refusal to expand scope before core stabilizes |
| Anti-drift thinking | Corrigibility thesis as architectural principle |
| Correction through reality | Operative truth arbitration; feedback as final validator |
| Preserve-over-chase | Refuse brute-force re-roll; restructure before retrying |
| Observer-first | Validators exist before the layer they validate |
| Closed-set discipline | Constitutional kernel: rules cannot be silently expanded |

## What ELIF does NOT inherit

- Manifest sovereignty over runtime artifacts (ELIF has no equivalent runtime manifest)
- Four-organ separation (OSG-specific architectural shape)
- Byte-frozen runtime path (ELIF has no equivalent; premature freezing would block development)
- Generation pipeline (LLM → IC → AV → country-shell → promotion)
- Closed-set enums for media observers, reviewer IDs, etc. (different objects)
- Stripe / commercial subscription concerns
- Phase numbering (P1, P2, ..., Phase 12+) — ELIF will need its own when one exists
- FR live formation untouchability — not applicable

## Operational rule

No mixing of OpenStudyGo's runtime concerns with ELIF's architecture discussions. No applying OpenStudyGo's phase numbering, IC/AV nomenclature, or generation cadence to ELIF. No suggesting ELIF should be implemented inside OpenStudyGo or vice versa.
