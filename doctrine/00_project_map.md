# Project Map

Four related systems exist in the operator's portfolio. They are distinct.

| System | Identity | Status |
|---|---|---|
| OpenStudyGo | Operational education and generation system | Live production (FR), stabilization mode |
| **ELIF** | **Corrigible discovery and validation architecture** | **Pre-prototype, one experimental data point** |
| AGORIA | Ontological layer informing ELIF | Conceptual; not a separate codebase for the current phase; no runtime doctrine until explicitly declared later |
| ORC | Future organic dynamic substrate | Deferred; not for current work |

This repo (`elif-lab`) covers ELIF only.

## Cross-project rule

Conversations about ELIF reference OpenStudyGo only as operational lineage or inherited governance principle. Conversations about OpenStudyGo reference ELIF only as conceptual descendant or future research direction. Neither system's decisions, doctrine, or roadmap flows into the other without explicit translation.
