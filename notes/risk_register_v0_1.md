# Risk Register — v0.1

**Date:** 2026-05-27

This document consolidates risks already named in the constitutional layer (risks 1-30) and the long-range roadmap (risks 31-39) into a single index. It does NOT add new risks. It improves operational hygiene by giving a single auditable surface.

If a new failure mode is identified through benchmark evidence, it is added here AND to the relevant doctrinal artifact. The doctrinal artifact remains the authoritative source.

## Severity legend

- **CRITICAL** — structural failure mode; activation triggers halt + operator review
- **HIGH** — significant drift risk; mitigation required at design time
- **MEDIUM** — degradation risk; periodic review sufficient
- **LOW** — meta-risk or speculative; awareness sufficient

## Maintenance discipline

This register is operational hygiene. It does NOT:
- Author new risks (those come from benchmark evidence)
- Modify doctrinal risks (they live in their doctrines)
- Add severity that isn't in the source
- Speculate about future risks beyond what's documented

It DOES:
- Index existing risks
- Provide single auditable surface
- Note cross-doctrine parallels
- Identify common detection mechanisms

If this register stops serving operational hygiene, it deprecates. Same rule as everything else.
